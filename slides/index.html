What is Git?

Files, locations, and views

Files

  • Git likes plain text files best
  • It can handle binaries (PDF, .doc, images), but not to its full potential
  • Lesson: Prefer open-source formats based in plain text (XHTML, HTML5, XML, Markdown, etc.

Locations

  • A Git repository lives in the folder with your project
  • It tracks all files in folders below that location
  • Lesson: be organized

Views

  • OS Gui (Windows Explorer/Mac Finder)
  • Command line/working tree
  • Git repository
  • Lesson: One file, one location, many views

The Forgotten Command Line

The Command Line

  • Simplest, most precise user interface
  • Also, to many users, the most daunting
  • Git is a command-line tool

Got Git?

  • Double-check; run 
    $ git --version
    (You don't type the dollar sign; that indicates your terminal/command prompt)
  • See http://help.github.com/ for installation instructions if you see anything but the Git version number, e.g., 
    git version 1.7.3.2

Configuring Git: It Wants Information

  • Your name
  • Your email address
  • To use color, if possible
  • To clean up after the mess Windows makes

Configuring Git: Commands

$ git config --global user.name "Your Name"
$ git config --global user.email "your@email.com"
$ git config --global color.ui "auto"
$ git config --global core.autocrlf "input"

Configuring Git: Check It

$ git config --global --list

Git in Your Workflow

Git in Your Workflow

  • Initialize or clone a repository
  • Only happens once per project

Git in Your Workflow

  1. Do stuff; File > Save
  2. Stage related stuff to be commited
  3. Make the commit (write to history)
  4. (Add more related stuff, commit...repeat...)
  5. Repeat...

Git in Your Workflow

  • Often: Make/change some files; save them
  • Often: "Stage" related changes to the index ($ git add <files>)
  • Next: Commit the staged changes: 
    $ git commit -m "Explanatory message"
